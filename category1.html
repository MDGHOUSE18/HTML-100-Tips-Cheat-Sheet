HTML Basics (Tips 1–10)

1. Semantic Tags 
<header>Header area</header>
<main>Main content</main>
<footer>Footer area</footer>

When/Why: Semantic tags improve SEO, accessibility, and make code easier to read.

2. The lang Attribute 
<html lang="en">

When/Why: Helps browsers, screen readers, and search engines understand the language of your page.

3. UTF-8 Charset 
<meta charset="UTF-8">

When/Why: Ensures your page correctly displays characters, emojis, and international text.

4. Viewport Meta 
<meta name="viewport" content="width=device-width, initial-scale=1.0">

When/Why: Makes your site responsive on mobile and tablets by scaling properly.

5. Favicon 
<link rel="icon" href="favicon.ico" type="image/x-icon">

When/Why: Adds a small icon in the browser tab, bookmarks, and search previews.

6. Alt Attribute on Images 
<img src="cat.jpg" alt="A cute cat">

When/Why: Essential for accessibility and SEO. Screen readers announce the description if the image can’t load.

7. Relative Paths 
<img src="images/logo.png" alt="Logo">

When/Why: Relative paths make your site portable across different environments (dev, staging, production).

8. Title Attribute 
<abbr title="HyperText Markup Language">HTML</abbr>

When/Why: Displays a tooltip on hover. Good for explaining abbreviations or extra info.

9. Meta Description 
<meta name="description" content="Learn 100 useful HTML tips and tricks">

When/Why: Appears in search engine results and improves SEO click-through rates.

10. Canonical Tag 
<link rel="canonical" href="https://example.com/tutorial.html">

When/Why: Prevents duplicate content issues by telling search engines the preferred page version.
