HTML Accessibility & ARIA (Tips 71–80)

71. ARIA Label 
<button aria-label="Close menu">✖</button>

When/Why: Provides an accessible label for elements without visible text.

72. ARIA Hidden 
<div aria-hidden="true">Invisible to screen readers</div>

When/Why: Hides decorative or redundant elements from assistive technologies.

73. Role Attribute 
<div role="alert">Error occurred!</div>

When/Why: Defines an element’s role (e.g., alert, button, navigation) for accessibility.

74. ARIA Live Region 
<div aria-live="polite">Status updates here</div>

When/Why: Announces changes dynamically to screen readers (chat messages, status updates).

75. Tabindex 
<div tabindex="0">Focusable Div</div>

When/Why: Makes non-interactive elements focusable with keyboard navigation.

76. Skip to Content Link 
<a href="#main" class="skip-link">Skip to content</a>
<main id="main">Main section</main>

When/Why: Improves navigation for keyboard and screen reader users.

77. Fieldset & Legend in Forms 
<fieldset>
  <legend>Payment Method</legend>
  <input type="radio" name="pay" id="card"> <label for="card">Card</label>
  <input type="radio" name="pay" id="paypal"> <label for="paypal">PayPal</label>
</fieldset>

When/Why: Groups related form controls with accessible labels.

78. Label with For Attribute 
<label for="username">Username</label>
<input type="text" id="username">

When/Why: Explicitly associates labels with form controls for accessibility.

79. ARIA Expanded 
<button aria-expanded="false" aria-controls="menu">Menu</button>
<ul id="menu" hidden><li>Item</li></ul>

When/Why: Indicates whether expandable content (dropdowns, accordions) is open or closed.

80. ARIA Describedby 
<input type="text" id="email" aria-describedby="emailHelp">
<small id="emailHelp">We'll never share your email.</small>

When/Why: Links additional descriptions or instructions to a form field.
