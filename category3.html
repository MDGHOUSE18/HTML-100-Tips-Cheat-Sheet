21. Placeholder 
<input type="text" placeholder="Enter your name">

When/Why: Provides a hint inside the input field until the user types.

22. Required Fields 
<input type="email" required>

When/Why: Ensures the field must be filled before submitting the form.

23. Pattern Validation 
<input type="text" pattern="[0-9]{3}-[0-9]{3}" placeholder="123-456">

When/Why: Validates input using regex. Great for phone numbers, zip codes, etc.

24. Email Input 
<input type="email">

When/Why: Browser validates for proper email format automatically.

25. Telephone Input 
<input type="tel">

When/Why: Opens numeric keypad on mobile devices for easier phone input.

26. URL Input 
<input type="url">

When/Why: Ensures a valid URL is entered.

27. Number Input with Min/Max 
<input type="number" min="1" max="10">

When/Why: Restricts input to a numeric range.

28. Step Attribute 
<input type="number" step="0.01">

When/Why: Controls step size for numeric inputs (useful for currency or decimals).

29. Autofocus 
<input type="text" autofocus>

When/Why: Automatically focuses on this input when the page loads.

30. Autocomplete Off 
<input type="password" autocomplete="off">

When/Why: Disables browser suggestions, useful for sensitive inputs.

31. Readonly 
<input type="text" value="You cannot edit this" readonly>

When/Why: Field is visible and selectable but cannot be changed.

32. Disabled 
<input type="text" value="Disabled field" disabled>

When/Why: Field is completely inactive and skipped when submitting.

33. Multiple Files 
<input type="file" multiple>

When/Why: Allows users to upload more than one file.

34. Size Attribute 
<input type="text" size="40">

When/Why: Controls the visible width of an input field.

35. Datalist Suggestions 
<input list="browsers">
<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
</datalist>

When/Why: Provides predefined suggestions as the user types.

36. Form Attribute 
<form id="signup"></form>
<input type="text" form="signup">

When/Why: Links inputs to a form even if placed outside the form tag.

37. Novalidate 
<form novalidate>
  <input type="email">
</form>

When/Why: Disables browser validation (useful when handling validation with JavaScript).

38. Spellcheck 
<textarea spellcheck="true"></textarea>

When/Why: Enables browser spell checking for text fields.

39. Inputmode 
<input type="text" inputmode="numeric">

When/Why: Optimizes the mobile keyboard (numeric, decimal, email, etc.).

40. Label For Accessibility 
<label for="email">Email:</label>
<input type="email" id="email">

When/Why: Improves accessibility by linking label text to its input field.
