HTML Text & Content (Tips 11–20)

11. <abbr> – Abbreviations 
<abbr title="Cascading Style Sheets">CSS</abbr>

When/Why: Adds a tooltip for abbreviations, improves accessibility, and helps readers understand acronyms.

12. <cite> – Citations 
<p>Written by <cite>Albert Einstein</cite></p>

When/Why: Marks up citations of books, articles, or authors. Helpful for semantic meaning.

13. <mark> – Highlighting 
<p>This is <mark>important</mark> text.</p>

When/Why: Highlights important content, often used to emphasize search results or key terms.

14. <time> – Dates & Times 
<time datetime="2025-09-03">September 3, 2025</time>

When/Why: Provides machine-readable dates for SEO, scheduling apps, and search engines.

15. <code> – Inline Code 
<p>Use <code>console.log()</code> in JavaScript.</p>

When/Why: Displays code snippets inline, great for documentation and tutorials.

16. <kbd> – Keyboard Input 
<p>Press <kbd>Ctrl</kbd> + <kbd>C</kbd> to copy.</p>

When/Why: Represents keyboard inputs, useful for tutorials or software guides.

17. <samp> – Program Output 
<p>System says: <samp>Access Denied</samp></p>

When/Why: Displays sample program output, making it distinct from normal text.

18. <q> – Short Inline Quotes 
<p>She said <q>HTML is fun!</q></p>

When/Why: Automatically adds quotation marks for inline quotes.

19. <blockquote> – Long Quotes 
<blockquote>
  <p>The only limit to our realization of tomorrow is our doubts of today.</p>
</blockquote>

When/Why: Used for longer quotes. Browsers often indent blockquotes for readability.

20. <address> – Contact Information 
<address>
  Contact: <a href="mailto:info@example.com">info@example.com</a>
</address>

When/Why: Defines contact details like email, address, or author info.
